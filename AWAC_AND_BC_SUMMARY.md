### 🎯 Что мы ожидаем от AWAC

**Гипотеза:** AWAC (advantage-weighted actor-critical) должен **улучшить** BC на сценариях отказов, особенно на тех, где нужна **переадаптация** и **closed-loop recovery**.

| Сценарий | BC результат | AWAC ожидание | Механизм улучшения |
|---|---|---|---|
| **normal** | ~72% | ~72–75% | Baseline, мало места для улучшения |
| **shift** (смещение 2–10 мм) | 40–50% | 55–70% | Closed-loop: переподвод и повторный захват |
| **rotate** (неправильный угол) | 25–40% | 45–60% | Адаптация к неудобной грани |
| **tcp_offset** (смещение TCP) | 30–40% | 50–65% | Коррекция калибровки в процессе |
| **friction** (скользкий товар) | 35–50% | 50–65% | Более осторожный контакт (contact_conf) |
| **vacuum_delay** (касание без захвата) | 40–60% | 60–75% | Раннее включение вакуума (contact_conf сигнал) |
| **vacuum_weak** (срыв при подъёме) | 30–50% | 50–70% | **Отличное место для AWAC**: recovery_mode + regrasp_bonus |

**Ключевой механизм:**
- BC политика уже знает "нормальный" путь (подвод → захват → укладка)
- AWAC поверх BC: онлайн взаимодействие показывает, какие **небольшие отклонения** приносят преимущество
- `A(s,a) = Q(s,a) − V(s)` определяет "хорошие" вариации
- **Advantage-weighted регрессия** усиливает те, которые находятся **в распределении данных**, но лучше

## AWAC Первый Прогон (v1, неудачный)

**Результат:** 0% на всех сценариях (детерминированная оценка).

**Причина:** Критическая ошибка инициализации.
- Актёр стартовал **полностью random** (нулевые веса MLP)
- Offline фаза: advantage-weighted регрессия пытается подогнать случайный актёр к демонстрациям, но без BC инициализации не может
- Результат: актёр остался близко к random, инференс — 0% успеха

**Урок:** AWAC нужна **хорошая инициализация** политики. Чистый AWAC from scratch не работает.

## AWAC Второй Прогон (v2, в процессе)

**Исправление:** BC инициализация перед AWAC.

```
5000 BC шагов → актёр выходит на 72–77% (как в train_bc_sac.py)
     ↓
30000 offline AWAC → критик учится на демонстрациях, актёр улучшается через AWR
     ↓
50000 online AWAC → взаимодействие среды, сценарии отказов, обновление политики
     ↓
EVAL → определяем успех BC vs AWAC на 8 сценариях × 50 эпизодов каждый
```

**Ожидание:**
- **Offline eval:** ~70–77% (BC держится)
- **Online success (в процессе обучения):** должен улучшиться с 24% (конец конца фазы) до 40–60%
- **Final eval на чистых сценариях:** AWAC ≥ BC (+5–15% на сложных)

## Ключевые Различия: AWAC vs BC

| Параметр | BC | AWAC |
|---|---|---|
| **Пайплайн** | Суперв. обучение на демонстрациях | BC + offline RL + online RL |
| **Актор обновление** | MSE к действиям (weighted) | Advantage-weighted log-prob (регуляризованное) |
| **Критик** | Нет | Есть (две Q-сети + target), обучится на буфере |
| **Online взаимодействие** | Нет | Да, сбор эпизодов, сценарии отказов |
| **Разовые отказы** | Не видит в обучении (отсеивает) | Видит online, учится восстанавливаться |
| **Сложность** | Низкая, быстро | Высокая, дольше |
| **Надёжность** | ~72–77% на clean, ~61% scenarios | Ожидание: ~75–80% clean, ~70+ scenarios |

## Интерпретация Результатов

### Сценарий 1: AWAC улучшил (ожидается)
```
vacuum_weak (срыв при подъёме)
BC: 40%  →  AWAC: 65%  (+25%)

Объяснение:
- BC политика обучена на успехах (отсеял отказы в сборе)
- AWAC видит отказы online, учится recovery_mode + regrasp_bonus
- closed-loop recovery выучивается через онлайн взаимодействие
```

### Сценарий 2: AWAC примерно равен BC (реалистично)
```
normal (без возмущений)
BC: 72%  →  AWAC: 73%  (+1%)

Объяснение:
- Нет отказов для обучения recovery
- AWAC может поправить мелкие несогласованности в BC
- Но основной сценарий уже хорошо покрыт BC
```

### Сценарий 3: AWAC деградировал (плохой знак)
```
shift (смещение товара)
BC: 45%  →  AWAC: 35%  (−10%)

Возможные причины:
- λ слишком низкий → актёр слишком свободен от данных
- Критик плохо откалиброван → даёт неправильные Q оценки
- Online data не помогает, помешает

→ Нужна чистка: λ ↑ или интернал данных online
```

## Литература и Мотивация

- **SAC from Scratch не обучается** → known problem (discrete local optima) → решение: демонстрации
- **BC + SAC деградирует** → SAC максимизирует Q при неправильной калибровке → решение: AWAC
- **AWAC (Advantage-Weighted Actor-Critic)** из "Learning from Demonstrations for Autonomous Navigation" (Nair et al.)
  - Гибрид BC (регуляризация к данным) + RL (преимущество-взвешивание)
  - Работает для manipulation, mobility, где данные ограничены
